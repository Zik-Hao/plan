# Компас — планировщик путешествий (PWA)

## Как запустить

Приложению нужен локальный веб-сервер (не открывайте `index.html` двойным
кликом — браузеры блокируют часть функций для файлов, открытых как `file://`,
в частности загрузку списка городов и офлайн-режим).

Проще всего:

```bash
cd travel-planner
python3 -m http.server 8000
```

и открыть `http://localhost:8000` в браузере. Подойдёт и любой другой простой
сервер (`npx serve`, расширение "Live Server" в VS Code и т.п.) — главное,
чтобы это было `http://` или `https://`, а не `file://`.

Для реального использования в поездке разверните папку на любом статическом
хостинге (GitHub Pages, Netlify, Vercel, Cloudflare Pages — подойдёт бесплатный
тариф любого из них) и откройте по `https://` ссылке — тогда «Добавить на
главный экран» установит приложение как полноценный офлайн-PWA.

## Как добавить новый город

1. Скопируйте `js/cities/spb.js` в `js/cities/<город>.js` и заполните своими
   местами по тому же образцу (`registerCity({...})`).
2. Откройте `js/cities/manifest.json` и добавьте имя нового файла в список:
   ```json
   ["spb.js", "kazan.js"]
   ```
   Без этого шага файл не подхватится — браузер не умеет сам «заглянуть»
   в папку и посмотреть, какие там лежат файлы, поэтому список городов
   ведётся явно в manifest.json.
3. Обновите страницу — новый город появится в выборе города при планировании
   маршрута.

## Экспорт и импорт маршрутов

В открытом маршруте — кнопка «⋮» в шапке:
- **Экспорт в PDF** — красиво оформленный документ с датами и заметками
  (поддерживает кириллицу).
- **Экспорт в файл (JSON)** — точная копия данных маршрута, самый надёжный
  вариант для локального бэкапа и переноса на другое устройство.

На экране «Мои маршруты» — кнопка «Импортировать маршрут» принимает оба
формата (PDF, экспортированный этим приложением, или JSON) и сама определяет,
какой перед ней файл.

## Офлайн-режим

При первом запуске (онлайн) приложение и все нужные ему библиотеки/шрифты
сохраняются в кэш браузера. После этого оно открывается и полностью работает
без интернета, включая экспорт/импорт PDF — до тех пор, пока вы не очистите
данные сайта в браузере.

## Структура проекта

```
index.html            — разметка всех экранов
css/style.css          — стили (Material Design)
js/registry.js         — реестр городов + автозагрузка файлов из js/cities
js/storage.js          — вся работа с localStorage (маршруты, места, категории)
js/pdf-export.js       — экспорт/импорт PDF
js/pdf-font.js         — шрифт с поддержкой кириллицы, встроенный в PDF
js/app.js              — основная логика интерфейса
js/vendor/             — локальная копия pdf.js (для чтения PDF при импорте)
js/cities/manifest.json — список файлов городов для автозагрузки
js/cities/spb.js       — данные по Санкт-Петербургу
sw.js, manifest.json    — офлайн-кэш и PWA-манифест
```
